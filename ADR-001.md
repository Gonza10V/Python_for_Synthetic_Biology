# ADR-001: Use a Quarto chapter-first book with executable Python examples

## Status
Accepted

## Context
The project needs a format that supports long-form technical writing, executable code, visible outputs, and easy web publication.

## Decision
Use Quarto as the primary publishing system and organize the book as a chapter-first repository with `.qmd` source files and executable Python cells.

## Options considered

### Option A: Quarto book
Pros:
- strong support for technical books
- executable code and outputs
- clean HTML publishing
- good cross references and navigation

Cons:
- requires learning Quarto conventions
- some customization may take extra effort

### Option B: Jupyter notebooks only
Pros:
- familiar for code-first work
- highly interactive

Cons:
- weaker long-form narrative structure
- harder to maintain as a coherent book
- harder to keep style and navigation consistent

### Option C: Static markdown site without execution
Pros:
- simple authoring model
- easy version control

Cons:
- no built-in executable examples
- weaker fit for a code-centered teaching book

## Why this option is best now
Quarto best matches the goal of teaching Python through executable, chapter-based synthetic biology examples.

## Consequences
- chapter files become the primary unit of authoring
- examples should remain small and reproducible
- helper code can be separated cleanly into `tools/`

## Risks and mitigations
- Risk: chapters become too notebook-like.
  - Mitigation: keep a consistent narrative structure.
- Risk: rendering breaks as dependencies grow.
  - Mitigation: keep examples lightweight and test renders often.

## Revisit triggers
- the project shifts to notebook-first teaching
- publishing requirements change significantly
- companion software becomes the primary deliverable
